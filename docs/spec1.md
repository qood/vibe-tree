# Claude Codeを活用したタスク分解、ブランチ・PR作成、状況視覚化、指示の抽象化を行うフロー

## 0. ゴール

- **外部ツール（Notion, Figma, GitHub Issueなど）のリンクを取り込み、事前情報をClaudeに共有**して、既存の整理済み内容を把握させる。
- **Claude Codeと対話しながらコードベースを確認し、タスクを分解**する。
- 分解したタスクごとに**ブランチを作成し、対応するIssueやプロジェクトを紐づけ**、PRを自動作成する。
- **各タスクのブランチに紐づくPRの状況やCIステータスをグラフで視覚化**し、状況を把握しやすくする。
- 最初にClaudeと練った**計画（指示書）をブランチごとにメモとしてDBに保存**し、後から参照できるようにする。
- **指示内容を抽象化・最適化して、同じ指示を繰り返さなくて済むようにする**。

## 1. フロー全体

### 1.1 外部リンク取り込みと事前情報共有

- ユーザーはWeb UI上でNotionやFigma、GitHub Issueのリンクを入力。
- Claude Codeはこれらのリンク先から情報を取得し、既存の整理済み内容を把握する。

### 1.2 タスク分解とブランチ作成

- Claude Codeと対話しながら、コードベースを確認しつつタスクを分解。
- 分解した各タスクに対して自動でブランチを切り、関連するIssueやプロジェクトを紐づける。
- 各ブランチごとに初期計画（指示書）をメモとしてDBに保存し、後から参照可能にする。

### 1.3 PR作成と状況視覚化

- 作成されたブランチから自動的にPRを作成。
- 各PRのリンク、ステータス、CI通過状況を取得し、Web UI上でタスク状況をグラフや一覧で視覚化。

### 1.4 指示の抽象化・最適化

- Claude Codeに出した指示をDBに蓄積。
- 同じ指示を繰り返さないよう、具体的な指示を徐々に**抽象化し、「この種別のファイルではこうする」といったルールを最適化**する。

## 2. 受け入れ条件

- 外部リンクの取り込みからClaudeへの情報共有ができること。
- タスク分解、ブランチ・PR作成、状況視覚化がスムーズに行えること。
- 指示内容が抽象化・最適化され、同じ指示の繰り返しが減ること。
